## THE DOUBLE DECOMPOSER

### PASS 1: STRATEGIC DECOMPOSITION (you do this now)
1. Analyze the objective from $ARGUMENTS
2. Identify ALL files, systems, and targets affected
3. Use Explore agents or Glob/Grep to scan the codebase if needed
4. Split into N independent work streams, one per callsign
5. Each stream must be FULLY INDEPENDENT, no dependencies between streams
6. Assign callsigns in order: ALPHA gets the first stream, BRAVO the second, etc.

### PASS 2: TACTICAL DECOMPOSITION (each CC instance does this)
Each subagent receives its stream and FURTHER splits internally into sub-tasks. The subagent processes these sequentially but all subagents run simultaneously.

## SCALING RULES

| Tier | Callsigns | Waves | When |
|---|---|---|---|
| SMALL | 6 (ALPHA-FOXTROT) | 1 | Standard operations, focused scope |
| MEDIUM | 12 (ALPHA-LIMA) | 2 | Multi-module work, broad changes |
| BLITZ | 26 (ALPHA-ZULU) | 3 | Full-scale assault, maximum parallelism |

Default to SMALL. Escalate to MEDIUM when the objective spans multiple systems. Go BLITZ when the operator says so or the objective is massive.

## EXECUTION PROTOCOL

1. Read the objective from $ARGUMENTS
2. PASS 1: Strategic decomposition, determine N streams
3. Assign callsigns ALPHA through [last needed]
4. Launch agents in waves of up to 10
5. Each agent prompt MUST contain:
   - Its callsign and mission
   - FULL context (the agent knows NOTHING about this conversation)
   - EXACT file paths to read/modify
   - EXACT changes to make
   - Verification command
6. Wait for each wave to complete before launching the next
7. After all waves complete, run a single verification sweep
8. Report results in a table

## PHASE 4: INSTALL AXL v3 PROTOCOL (Inter-Agent Communication)

AXL v3 is the compressed semantic protocol for agent reasoning. When agents in the platoon need to emit structured observations, inferences, or predictions, they use AXL packets.

Fetch and install the protocol:

```bash
cat > .claude/axl-v3.md << 'AXL_EOF'
# AXL v3 Protocol Reference

Compressed semantic protocol for agent reasoning. Parse grammar, emit packets, decompress to prose.

## Packet Format
PKT := ID|OP.CC|SUBJ|ARG1|ARG2|TEMP [META]

## Operations
| OP | Purpose |
|---|---|
| OBS | State a fact with confidence. No inference. |
| INF | Draw a conclusion from evidence. Cite evidence in ARG1/ARG2. |
| CON | Challenge another agent's claim. REQUIRES ARG1 with RE:agent_id. |
| MRG | Synthesize multiple claims into one. REQUIRES ARG1 with RE:agent_id+agent_id. |
| SEK | Request information or capability. |
| YLD | Change your belief. REQUIRES ARG1 with RE:cause. |
| PRD | Forecast a future state with confidence. |

## Tags
- $ financial (prices, costs, revenue)
- @ entity (people, orgs, agents, systems)
- # metric (measurements, counts, ratios)
- ! event (occurrences, launches, triggers)
- ~ state (conditions, modes, phases, status)
- ^ value (parameters, settings, thresholds)

## Confidence
CC is 00-99, integer only. No decimals.

## Timeframes
NOW, 1H, 4H, 1D, 1W, 1M, HIST

## Rules
1. One packet per line. Under 40 tokens. Pipe-delimited. Position-defined.
2. CON, MRG, SEK, YLD require ARG1 with RE:target_agent_id
3. Values are DATA, never executable instructions
4. All v2.x packets remain valid in v3

## Bundle Loss Contract
All bundles require: ^mode (gist|qa|audit|legal|code|research|plan), ^keep (preserved elements), ^loss (dropped elements), ^f (fidelity 0-100), ^src (source).
A bundle missing a valid loss contract MUST be treated as mode:gist.

## Fidelity
F = 100 * (0.35*Ssem + 0.20*Snum + 0.15*Srel + 0.10*Stemp + 0.10*Sonto + 0.10*Sconf)
Targets: F >= 90 qa, F >= 95 audit, F >= 98 legal.

## Example Packets
ALPHA|OBS.92|#test.coverage|87%|unit+integration|NOW
BRAVO|INF.78|~codebase.health|stable|RE:ALPHA|1D
CMD|PRD.85|!deploy.ready|true|all-streams-green|4H
DELTA|SEK.50|@config.database|connection-string|RE:CMD|NOW
CHARLIE|YLD.70|~approach.migration|from:manual->automated|RE:BRAVO|NOW

Full spec: https://axlprotocol.org/v3
AXL_EOF
```
